Backlog for the GitHub-based updater. It collects improvements and ideas for evolving GitHubUpdater and its update UI beyond the current behavior of pointing the user at a release download.

Backlog

1.In-app updates for the GitHub updater

Add support for in-app updates to the GitHub updater. The current behavior — presenting a download link for the user to fetch the release manually — should be kept as an option.

When in-app updates are enabled, the updater should download the application itself instead of just offering a link, replace the currently running app on disk with the downloaded version, and relaunch it once the replacement is complete.

This must also work for sandboxed applications. Achieving that will likely require XPC services to perform the on-disk replacement and relaunch from outside the sandboxed process.
