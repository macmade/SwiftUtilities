GitHubUpdater SwiftUI Update Window

Generated · 2026-06-27

This roadmap covers reworking how GitHubUpdater reports an available update. Today the "update available" outcome is shown as a plain NSAlert with View / Later buttons that opens the release page in the default browser. The work below replaces that single outcome with a dedicated, SwiftUI-based update window that presents the release notes and richer actions. The up-to-date and error outcomes are out of scope and keep their existing alerts.

  1. Custom update window

    Replace the NSAlert used for the "update available" outcome with a dedicated window. Only this outcome adopts the new window; the up-to-date and error outcomes continue to use their existing modal alerts.

  2. Release notes with Markdown rendering

    Display the release notes in the window, rendering the release's Markdown body so headings, lists, links, and other formatting appear as intended rather than as raw text. This requires carrying the release notes through the update check, beyond the release page URL the updater captures today.

  3. Download, View on GitHub, and Later actions

    Offer three buttons. Download opens the release's direct download link in the user's default browser, letting the browser handle the download. View on GitHub opens the release page (html_url) in the default browser. Later dismisses the window without acting.

  4. Built in SwiftUI

    Implement the update window and its contents in SwiftUI, hosted appropriately for the AppKit-based updater, keeping the platform guards consistent with the existing GitHubUpdater+AppKit code.
